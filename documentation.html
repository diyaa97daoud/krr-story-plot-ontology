Story Plot Ontology Documentation

This documentation describes a modular OWL/Turtle knowledge model for story plots, aligned with the KRR homework instructions.

Design Goals

  • Represent characters, places, events, actions, and plot structure.
  • Support timeline reconstruction through temporal relations.
  • Enable consistency checks (for example, impossible co-location situations).
  • Keep examples in a separate instance file.

Modules

Provided/fixed modules: fo.ttl, meo.ttl, rcc.ttl

New modules: model-social.ttl, model-place.ttl, model-time.ttl, model-event.ttl, model-action.ttl, model-plot.ttl

Master import ontology: model-story.ttl

Instance example: instances-narn-lite.ttl

Modeling Choices

Characters and Relations

Character typing is connected to the Middle-earth classes where relevant. Family and social relations are represented with explicit inverse and symmetric properties when needed.

Places and Space

Places are subclasses of rcc:Region and can be organized through transitive containment. RCC8 relations are used for topological constraints.

Events and Time

Events carry participants and locations. Ordering between events is represented through transitive beforeEvent. A separate time module provides Allen-style relation vocabulary.

Plot Structure

Narrative abstractions (story, episode, plot point, climax, resolution) are separated from event internals and linked with dedicated properties such as realizedByEvent and hasEndingEvent.

Representative Vocabulary and Examples

Social Module

Example terms include friendOf, allyOf, enemyOf, mentorOf, and protects. In the instance file, Thingol mentors and protects Turin, while Beleg is modeled as both ally and friend.

Place Module

Places are typed as Settlement, Fortress, or WildArea, and connected through partOfPlace, containsPlace, adjacentPlace, and RCC8 relations. For example, Cabed Naeramarth is part of Brethil and Brethil contains Cabed Naeramarth.

Time Module

Temporal entities are now explicitly instantiated and linked to events through hasTemporalEntity. Example relations include before and during between named periods such as the Doriath fosterage period and the council moment within it.

Event Module

Event examples cover BattleEvent, JourneyEvent, CouncilEvent, and DeathEvent. The Doriath journey has a modeled sub-event for counsel, and all core events are linked to both a place and a temporal entity.

Action Module

Actions are distinct from events. The instance file illustrates UseItemAction, CombatAction, SpeechAction, and MoveAction, each linked to an agent, event context, place, and optional target or instrument.

Plot Module

The story is organized with episodes, plot points, a climax, a resolution, and an ending event. These narrative entities are connected to the underlying events through eventInEpisode, realizedByEvent, and hasEndingEvent.

Competency Questions

Query templates are provided in competency-questions.md and can be executed against instances-narn-lite.ttl.

Validation Notes

The instance file contains a small hypothetical validation pattern for impossible co-location: Turin is attached to two disconnected places at the same temporal entity in test events. This is not part of the narrative canon; it exists to demonstrate how competency questions can detect a spatio-temporal inconsistency pattern.

Reasoner validation was performed in Protégé. HermiT completed on the full ontology without errors. ELK also loaded the ontology, but reported expected incompleteness warnings because the model uses OWL constructs outside the OWL 2 EL profile, notably inverse object properties, disjoint object properties, inverse property expressions, and qualified minimum cardinality restrictions. These ELK warnings reflect profile limitations, not a modeling failure.
